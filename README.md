# نظام رقمنة الوثائق بالأقسام — Flask + MySQL + (Gemini / Claude)

نظام ويب بتسجيل دخول: تنشئ **أقساماً**، كل قسم له **قالب إكسل ناتج** ترفعه مرة
واحدة، ثم ترفع ملفات PDF داخل القسم فيستخرج Gemini نصوص الصفحات، ويعبّأ القالب
تلقائياً، وتنزّل **ملف إكسل جاهزاً بنفس شكل قالبك** — سهل التعديل.

## الفكرة
- **قسم = نوع وثائق + قالب إكسل.** ترفع قالباً (تسميات الحقول في العمود A،
  والقيم ستُكتب في B، وأي جدول بصف ترويسات). النظام يتعلّم بنيته تلقائياً.
- إضافة نوع جديد = إنشاء قسم جديد برفع قالبه. **بلا أي تعديل كود.**
- محرك تعبئة القالب قابل للتبديل من `.env`:
  `STRUCTURE_ENGINE=gemini` (يعمل الآن بمفتاح Gemini وحده) أو
  `STRUCTURE_ENGINE=claude` عندما توفّر مفتاح Anthropic (أدق في التدقيق).

## التشغيل محلياً / على VPS
```bash
python -m venv venv && source venv/bin/activate   # ويندوز: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env    # ثم عدّل القيم
python app.py           # للتجربة
# للإنتاج:
gunicorn -c gunicorn.conf.py wsgi:app
```
الدخول الافتراضي: `admin / admin123` — **غيّره في `.env` فوراً.**

## سير الاستخدام
1. سجّل الدخول → صفحة **الأقسام**.
2. أنشئ قسماً: اسم + قالب إكسل (مثل قالب «عقد بيع عقاري»).
3. ادخل القسم وارفع PDF → يقسَّم لصفحات، يستخرج Gemini نصوصها،
   ثم يعبّأ القالب ويُنتج ملف إكسل.
4. من صفحة الوثيقة: راجع القيم والحقول المشكوك فيها، ونزّل الإكسل.

## ملاحظات
- MOCK=1 يشغّل كل شيء ببيانات وهمية بلا مفاتيح — للتجربة قبل الشراء.
- تُنشأ الجداول (users/sections/documents/pages) تلقائياً عند أول تشغيل.
- إن أضفت أعمدة جديدة للجداول لاحقاً على قاعدة قديمة، احذف الجداول أو أضف
  الأعمدة يدوياً (SQLAlchemy لا يعدّل جداول موجودة).
